# 多段コードレビュー

サブエージェントを使った3段階レビューを実施します。

## レビュープロセス

### Stage 1: 仕様準拠レビュー
- 変更が元の要求を満たしているか
- 不足している機能はないか
- 過剰な変更（スコープクリープ）はないか

### Stage 2: コード品質レビュー
- TypeScript型安全性
- エラーハンドリングの適切さ
- パフォーマンスの懸念（特にクエリ性能）
- テストの有無と十分性

### Stage 3: データ安全性レビュー（keiba固有）
data-safety-reviewer エージェントを使って以下をチェック:
- DELETE文にWHERE句があるか
- libsql:// を使っていないか
- db.batch() を使っていないか
- beforeDate フィルタが予測クエリに適用されているか
- Turso接続がHTTPSか

## 実行方法

1. `git diff` で変更内容を取得
2. 各ステージを順にサブエージェントで実行
3. 問題があれば具体的な修正案を提示
4. 全ステージパスなら「レビュー完了: 問題なし」と報告
