# ブレインストーミング

ラフなアイデアを構造化された検討に変換します。

## プロセス

1. **現状把握**: まず関連データを分析する（的中率、ROI、SHAP値等）
2. **一度に一つの質問**: 選択式で聞く（はい/いいえ、A/B/C形式）
3. **常に2〜3のアプローチを提案**: 各案のメリット・デメリットを明示
4. **YAGNI原則**: 不要な複雑さを排除。「やれること」ではなく「やるべきこと」を提案
5. **仮説レベルの明示**: 根拠が薄いときは正直に「これは仮説レベル」と伝える

## CLAUDE.md準拠チェック（必須）

提案の前に以下を確認する:
- AI単独実力の計測は済んでいるか？（オッズ除外モデルのTop-1/ROI）
- 市場依存度は定量化されているか？（全特徴量 vs オッズ除外の差分）
- AI付加価値がゼロ以下なら → 特徴量追加ではなくモデル構造の見直しが先

## 出力フォーマット

```
### アイデア: [タイトル]
- 仮説: [何が改善されるか]
- 根拠: [データ/分析結果]
- 信頼度: [高/中/低 — 仮説レベルなら明示]
- 実装コスト: [小/中/大]
- 期待効果: [的中率+X%、ROI+Yポイント等の具体値]
- リスク: [副作用、データ要件等]
```
