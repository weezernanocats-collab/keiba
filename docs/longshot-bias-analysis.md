# 競馬予測システム：推奨ベットのロングショットバイアス問題

## 用語説明

### 期待値（EV: Expected Value）

1回のベットで「平均的にいくら戻ってくるか」を示す指標。

```
期待値 = 的中確率 × オッズ

例: 的中率30%、オッズ4倍の馬
  → 0.30 × 4 = 1.20（100円賭けると平均120円戻る → 20円の利益）

例: 的中率3%、オッズ50倍の馬
  → 0.03 × 50 = 1.50（100円賭けると平均150円戻る → 50円の利益）
```

EV > 1.0 なら「長期的にはプラス」、EV < 1.0 なら「長期的にはマイナス」。
ただし「長期的」とは数百〜数千回の試行が前提。

### Kelly基準（Kelly Criterion）

「手持ち資金の何%を賭けるのが最適か」を計算する公式。的中確率とオッズから導く。

```
Kelly = (的中確率 × オッズ - 1) ÷ (オッズ - 1)

例: 的中率30%、オッズ4倍
  → (0.30×4 - 1) ÷ (4-1) = 0.2 ÷ 3 = 0.067（資金の6.7%を賭けるのが最適）

例: 的中率3%、オッズ50倍
  → (0.03×50 - 1) ÷ (50-1) = 0.5 ÷ 49 = 0.010（資金の1.0%が最適）
```

的中率が低いと、EVがプラスでもKellyは小さくなる = 「儲かるけど少額にしておけ」という意味。

## 課題

**おすすめベットが高オッズ（穴馬）に偏りやすい**

現在の推奨ロジックは**期待値の高い順**に並べている。
すると以下のような逆転が起きる。

| 例 | 的中率 | オッズ | 期待値 | Kelly | 実感 |
|----|--------|--------|--------|-------|------|
| 本命馬A | 30% | 4倍 | 1.20 | 6.7% | 3回に1回当たる。安定 |
| 穴馬B | 3% | 50倍 | 1.50 | 1.0% | 33回に1回。ほぼ外れ続ける |

→ **穴馬BのほうがEVが高い**ので上位に推奨される。
→ しかし個人が月に数十レースしか買わないなら、33回に1回の的中は体験できない可能性が高い。

**要するに「計算上は儲かるはずだが、現実の試行回数では収束しない」問題。**

## 現状のセーフガード

| 対策 | 内容 | 効果 |
|------|------|------|
| オッズ範囲制限 | 3〜50倍のみ対象 | 超大穴（100倍超）は除外 |
| Kelly > 0.02 | 最適賭け比率が極小のものは除外 | 的中率が極端に低いものを排除 |
| edge > 10% | モデル確率とオッズの乖離が小さいものは除外 | 微妙なものを排除 |
| ダート短距離除外 | モデル精度が低いカテゴリを除外 | 不得意領域の推奨を抑制 |

→ ある程度は機能しているが、「50倍 × 3% = EV 1.50」のようなケースは通り抜ける。

## 対策案

### 案A：最低的中率フィルタ（シンプル）

**やること**: 的中率が一定以下のベットは推奨しない

```
条件追加: 的中率 >= 5%（単勝）/ >= 10%（複勝）
```

| 項目 | 内容 |
|------|------|
| メリット | 実装が簡単。低的中率の穴馬推奨が消える |
| デメリット | 閾値の根拠が恣意的。本当に儲かる穴馬も切り捨てる |
| 向いている人 | 安定志向。的中率を重視する人 |

### 案B：信頼度重み付き期待値（バランス型）

**やること**: 的中率が低いほど期待値を割り引く補正をかける

```
調整後の期待値 = 期待値 × √(的中確率)

穴馬B: 1.50 × √0.03 = 0.26（大幅ダウン）
本命A: 1.20 × √0.30 = 0.66（少しダウン）
→ 本命Aが上位に来る
```

| 項目 | 内容 |
|------|------|
| メリット | 穴馬と本命のバランスが取れる。閾値不要で滑らかに補正 |
| デメリット | 補正関数（√か logか等）の選択根拠が必要 |
| 向いている人 | 的中率と回収率のバランスを取りたい人 |

### 案C：Kelly基準でソート（理論的に最適）

**やること**: 期待値の代わりにKelly（最適賭け金比率）で並べる

```
穴馬B: Kelly = 0.010（資金の1.0%） → 低順位
本命A: Kelly = 0.067（資金の6.7%） → 高順位
```

| 項目 | 内容 |
|------|------|
| メリット | 賭け理論として最も合理的。的中率が低いと自然に順位が下がる |
| デメリット | モデルの予測確率が少しズレるとKellyが大きく変わる（精度依存） |
| 向いている人 | 理論派。モデル精度に自信がある場合 |

## 判断のポイント

1. **ユーザーは月に何レース買うか** → 少ないなら穴馬推奨は危険（案A or C向き）
2. **的中率と回収率どちらを重視するか** → 的中率なら案A、回収率なら現状維持
3. **モデルの低確率帯の精度** → 「3%」という予測が信頼できるか（できないなら案A）
4. **併用も可能** → 例：Kelly順でソート + 的中率5%未満に警告ラベル表示
