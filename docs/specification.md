# KEIBA MASTER 機能仕様書

> **Last updated**: 2026-04-02

## 概要

競馬予想Webアプリケーション。netkeibaからスクレイピングしたデータを基に、機械学習モデル（CatBoost/XGBoost）で各レースの予想を自動生成し、成績を追跡する。

---

## 1. 予想生成

### 1.1 自動生成フロー

1. Vercel Cron / GitHub Actions が `/api/sync` にPOST
2. 当日のレース一覧をnetkeibaから取得
3. 各レースの出馬表・馬詳細・オッズを取得
4. `prediction-engine.ts` で予想を生成:
   - 17ファクタースコアリング → ベイズ推定 → 馬場バイアス補正
   - XGBoost+CatBoostアンサンブル推論 (53特徴量)
   - No-Oddsモデル推論 (42特徴量) → AI推奨買い目生成
   - 市場オッズブレンド → 最終ランキング
5. `predictions` テーブルに保存

### 1.2 予想の出力

| 項目 | 内容 |
|---|---|
| **ブレンド予想** | Top 6推奨馬（◎○▲△×☆）+ スコア + 脚質 |
| **AI推奨買い目** | 馬連/ワイドの組み合わせ（一強/二強/三つ巴/混戦パターン） |
| **信頼度** | 0〜100%（スコア差40点 + データ充実度35点 + 統計15点） |
| **分析テキスト** | レース展開予想、キーファクター |
| **推奨馬券** | Value Bet検出（Kelly Criterion） |
| **市場比較** | AIと市場の乖離分析 |

### 1.3 バイアス再計算

- 同日・同競馬場で3R以上の結果が確定すると馬場バイアスが算出可能
- 予想詳細ページの「バイアス更新」ボタンで手動再計算
- API: `GET /api/predictions/{raceId}?biasUpdate=1`

---

## 2. 成績追跡

### 2.1 レース結果照合

レース終了後、17:00 Cronで結果を自動取得し、予想と照合:
- 単勝的中: 本命馬（◎）が1着
- 複勝的中: 本命馬が3着以内
- ROI: 単勝100円想定 × 実オッズ

### 2.2 AI推奨買い目の成績追跡

`/api/accuracy-stats` で以下を集計:

- **券種別成績**: 馬連/ワイドそれぞれのベット数・的中数・的中率・投資額・回収額・ROI
- **累積推移**: 日ごとの累積利益・ROI・的中率
- **的中判定ロジック** (`isBetHit`):
  - 馬連: 選択した2頭が両方とも上位2着以内
  - ワイド: 選択した2頭が両方とも上位3着以内
  - BOX買い: 複数の組み合わせそれぞれを個別判定

## 3. ページ仕様

### 3.1 トップページ (`/`)

- 当日のレース一覧
- 最近の的中情報
- 統計サマリー（的中率、ROI）

### 3.2 AI予想ページ (`/predictions`)

**タブ構成:**
- **AI予想タブ**: 今後のレース一覧
- **過去予想タブ**: 過去の予想結果（的中率・ROI）

**AI予想タブ:**
- 日付でグループ化
- 各レース行: レース番号 / グレードバッジ / レース名 / 発走時刻 / 競馬場 / コース情報 / 信頼度バッジ / お気に入りボタン
- 信頼度フィルター（全信頼度/高70%+/中50-69%/低<50%/未算出）
- タップで展開:
  - 予想印（◎○▲△×☆）
  - AI推奨買い目（パターン + 馬連/ワイド + 信頼度ラベル）
  - 「詳細を見る →」リンク

### 3.3 予想詳細ページ (`/predictions/[raceId]`)

**セクションナビ（スティッキー）:**
答え合わせ | AI買い目 | サマリー | ブレンド予想 | 詳細分析

**各セクション:**

1. **答え合わせ**（結果確定時のみ表示）
   - 的中/不的中表示
   - 実際の着順 vs AI予想の比較

2. **AI推奨買い目**
   - パターンバッジ（一強/二強/三つ巴/混戦）
   - 馬連/ワイドの推奨組み合わせ
   - 信頼度ラベル（本線/押さえ）
   - 各馬のAIランク + AI予測確率

3. **サマリー**
   - 分析テキスト
   - 信頼度ゲージ
   - 出走頭数
   - バイアス更新ボタン（同日3R+結果確定時に表示）

4. **ブレンド予想**
   - Top 6推奨馬の詳細カード（馬番/馬名/騎手/スコア/確率/脚質）
   - 確率バー表示

5. **詳細分析**（`<details>` で折りたたみ）
   - レース分析: ペース予想、展開シナリオ
   - 馬券戦略: 推奨買い方
   - 市場比較: AI評価と市場オッズの乖離
   - AI独自推奨: 単勝/複勝の独自ピック
   - AI単独評価: No-Oddsモデルの順位表
   - 推奨馬券: EV計算結果
   - 期待値+: 高EV馬券
   - シミュレーション: モンテカルロ結果

### 3.4 的中率分析ページ (`/stats`)

**AI推奨買い目の成績セクション:**
- StatCardグリッド: 対象レース数 / 総ベット数 / 券種別的中率
- 成績テーブル: 券種 / ベット数 / 的中 / 的中率 / 投資額 / 回収額 / ROI / 収支
- 累積推移グラフ（3日以上のデータ時）: 利益 / ROI / 的中率の折れ線

**その他:**
- 全体統計（的中率、ROI）
- 信頼度帯別分析
- 馬券種別成績
- 競馬場別統計
- ROI推移グラフ

### 3.5 レース詳細ページ (`/races/[raceId]`)

- 出馬表（枠番/馬番/馬名/騎手/斤量/オッズ）
- オッズ取得/更新ボタン
- 予算シミュレーター
- モンテカルロシミュレーション

### 3.6 馬詳細ページ (`/horses/[horseId]`)

- 基本情報（血統、調教師、成績）
- 過去成績グラフ（着順推移）
- コンディション分析

### 3.7 騎手詳細ページ (`/jockeys/[jockeyId]`)

- キャリア統計（勝率、複勝率）
- 直近フォーム
- 距離別・コース別成績

### 3.8 管理画面 (`/admin`)

| 機能 | 説明 |
|---|---|
| フル同期 | 指定日のデータ一括取得 + 予想生成 |
| 予想再生成 | 結果取得 → バイアス算出 → 予想再生成 |
| 結果一括取得 | 当日の全レース結果を一括取得 |
| バルクインポート | 日付範囲指定で過去データ一括取込 |
| 一括照合 | 未照合レースの予想vs結果を一括評価 |
| ウェイト校正 | ファクター重みの自動キャリブレーション |

## 4. データベーススキーマ

### 4.1 主要テーブル

| テーブル | 主キー | 概要 |
|---|---|---|
| `racecourses` | id | 競馬場マスタ |
| `races` | id | レース情報（日時、条件、グレード） |
| `race_entries` | id | 出走馬（馬番、騎手、斤量、着順、オッズ） |
| `horses` | id | 馬マスタ（血統、戦績） |
| `jockeys` | id | 騎手マスタ |
| `past_performances` | id | 過去成績（タイム、着順、上がり3F） |
| `odds` | id | オッズデータ（単勝、複勝） |
| `odds_snapshots` | id | オッズスナップショット（時系列） |
| `predictions` | id | AI予想（picks, analysis, bets） |
| `prediction_results` | id | 予想評価結果（的中判定、ROI） |
| `horse_traits` | id | 馬の特性（長所/短所） |
| `scheduler_runs` | id | スケジューラ実行ログ |

### 4.2 主要カラム

**races:**
- date, time, raceNumber, name, grade
- trackType (芝/ダート/障害), distance, trackCondition, weather
- status (予定/出走確定/結果確定/中止)
- lap_times_json, pace_type

**predictions:**
- race_id, confidence, summary
- picks_json: Top 6推奨馬
- analysis_json: 分析データ（horseScores, winProbabilities, marketAnalysis, aiRankingBets）
- bets_json: 推奨馬券

**prediction_results:**
- prediction_id, race_id
- top_pick_position: 本命馬の着順
- top3_hits: Top3の的中数
- roi: 回収率
- bet_hit_types: 的中した馬券種別

## 5. 外部連携

### 5.1 netkeiba.com（データソース）

- スクレイピングで全データを取得
- レート制限: 約2,600req/12h（1,200ms間隔）
- エンコーディング: EUC-JP（一部UTF-8）

### 5.2 Turso（データベース）

- LibSQL (分散SQLite)
- HTTPS接続必須
- batch() 使用禁止（障害対策）

### 5.3 Vercel（ホスティング）

- Hobbyプラン
- Functions: 60秒タイムアウト
- Cron: 6ジョブ（UTC指定）

### 5.4 GitHub Actions（自動化）

- モデル再学習（毎週月曜）
- データ取得・予想生成（毎日）
- 結果取得・照合（毎日17:00）
